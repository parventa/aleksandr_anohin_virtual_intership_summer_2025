**Проект: "Система управления медицинским центром"**

**Описание:**

Цель этого проекта - создать систему управления медицинским центром.
Эта система будет включать информацию о пациентах, врачах,
предоставляемых услугах, регистрации приемов и медицинской
истории пациентов.

**Основные этапы проекта:**

1. **Проектирование базы данных:**
Создайте ER-диаграмму, которая отображает отношения между
сущностями "Пациенты", "Врачи", "Услуги", "Приемы" и
"Медицинская история".

2. **Создание базы данных:**
Используйте SQL DDL для создания базы данных. Она должна
содержать следующие таблицы:
   - Таблица "Пациенты": ID пациента, Имя, Фамилия, Дата рождения,
     Контактная информация.
   - Таблица "Врачи": ID врача, Имя, Фамилия, Специализация.
   - Таблица "Услуги": ID услуги, Название услуги, Описание, Стоимость.
   - Таблица "Приемы": ID приема, ID пациента, ID врача,
     ID услуги, Дата и время приема.
   - Таблица "Медицинская история": ID записи, ID пациента,
     Дата, Описание диагноза, Назначенное лечение.

3. **Создание внешних ключей и индексов:**
Убедитесь, что все отношения между таблицами правильно заданы
с использованием внешних ключей. Также создайте индексы для
улучшения производительности запросов.

4. **Создание ограничений:**
Разработайте и примените ограничения целостности данных,
такие как CHECK (например, стоимость услуги не может быть отрицательной),
UNIQUE и NOT NULL.

**Ожидаемые результаты:**

По окончании проекта у вас должна быть функциональная база данных,
которую можно использовать для управления медицинским центром.
Это поможет эффективно управлять ресурсами центра, предоставлять
быстрый доступ к информации о пациентах и услугах, а также следить
за историей заболеваний пациентов.


Напишите миграционный скрипт с SQL командами,
который создаёт базу данных с нуля.

Данный скрипт должен включать в себя как минимум следующие команды:
- CREATE DATABASE;
- CREATE TABLE (+ PRIMARY KEY, FOREIGN KEY, NOT NULL);
- ALTER TABLE ADD COLUMN;
- CREATE INDEX (UNIQUE INDEX);
- CREATE VIEW;
- CREATE ROLE;
- CREATE USER;
- GRANT;

Продемонстрируйте возможности сохранения данных в созданную вами
структуру базы данных при помощи SQL скрипта с командами вставки данных.

Создайте в базе данных набор VIEW, которые помогут бизнесу с нахождением
ответом на типичные запросы и анализом данных хранящихся в базе данных.
